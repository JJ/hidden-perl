Perl and Git

Big and close friends

# from your .git/hooks/prepare-commit-message.sample

COMMIT_MSG_FILE=$1

/usr/bin/perl -i.bak -ne 'print unless(m/^. Please enter the commit message/..m/^#$/)' "$COMMIT_MSG_FILE"
You have probably lots of copies of this snippet of code in your hard dist, because it's included with every default git installation. It exemplifies the expressivity, handling of regular expressions, and integration with system tools that is exclusive to Perl

Let's break it down

COMMIT_MSG_FILE=$1 # Define environment variable from shell argument

/usr/bin/perl # Use system perl
  -i.bak      # Write the original file with .bak extension
  -ne         # Run the following script with while (<>) {} wrapped
  'print
     unless( # Opposite of if
       m/^. Please enter the commit message/ .. m/^#$/) # Between two regexes
   ' "$COMMIT_MSG_FILE" # applies to the file above
I had to look that up... it's probably not the easiest, but one you can check yourself. There are several more examples in the same directory and even in the same file. Most important lesson is in line three, however. Git assumes that you have a system Perl installed. No examples in Python or Ruby. 6.4% of git is Perl. But that's your local repo. Use fatpack

Do you have that in GitHub actions?

jobs:
  run_tests:
    runs-on: ubuntu-latest
    steps:
      - name: checkout
        uses: actions/checkout@v2
      - name: Install modules by apt
        run: sudo apt-get install libxml-rss-perl libmp3-info-perl
      - name: Prepare for tests
        run: |
          perl Makefile.PL
          make
      - name: Perform tests
        run: make disttest
Pretty straightforward, huh? It's as simple as using shell, which is obviously also supplied

Use Perl as a shell

In fact, GitHub::Actions is a mini-library that simplifies using GitHub actions command in Perl... You need either to install it in advance (fast, no dependencies) or fatpack it

Use system modules

use CPAN::Meta::YAML;
for my $f (split(";",$ENV{'yaml_files'})) {
  open my $fh, "<:utf8", $f;
  my $yaml_text = do { local $/; <$fh> };
  my $yaml = CPAN::Meta::YAML->read_string($yaml_text)
    or die CPAN::Meta::YAML->errstr;
  close $fh;
  print "✓ YAML «$f»";
}
The key here is to use an existing YAML module, which is rather lenient in most cases, instead of using an external utility. Many modules, mainly dependencies of testing ones, related to Debian packaging, or to CPAN (like this one), are already installed. Leveraging them will save you time packaging or creating any other solution to employ some module. Also, the preferred way of sending info to GitHub actions is via environment variables. That way, you can run them directly from a pipe or include them in the body of a github action step as above.

Automate Docker Hub builds or override them via hooks

Proceed only if you need to: post_checkout

#!/usr/bin/env perl

use Git;
my $repo = Git->repository (Directory => '.');

my @modified_files = $repo->command('diff', "--name-only", "HEAD", "HEAD^");

die("No Dockerfile modified in the last commit\n")
    unless grep( /Dockerfile/, @modified_files);
This script works as is (0 dependencies), because it's using the Git module that comes with git, and the perl that's installed in the runner. No dependencies. Impossible to do that with any other language.
